Browser compatibility

Our default browser was Google chrome. But we also tested the files using Microsoft Edge and Opera. The tests were done using MacOS and Windows.

All browsers showed similar behaviours. The links work perfectly. The whole page functions as desired. Both css and html were rendered displayed the same way using Google chrome, Opera and Microsoft Edge.

The page was tested in almost all sizes the browsers has. But we focused on (Fullscreen, iPhone X, iPad, iPad pro, Galaxy S9+, Galaxy Tab and responsive mode) While testing our page with different screen sizes, the page on Chrome and Microsoft Edge was not really responsive and the tables did not fit in the page. So an important part of the page was not visible. On Opera the tables were fully dispalyed when the screen gets smaller but the header and the footer were not complete. We realised that the main issue was the tables, because all other elements were dispalyed perfectly in all different screen sizes.

We fixed that this way: We first determined different screen sizes using @media. So that when the screen gets small, the tables will be structured vertically. Using this technique all elements fitted nicely and the page became fully responsive.

While printing the page or saving it as a PDF-file, browsers acted differently. Google Chrome and Microsoft Edge deleted all background colours and background images. And the page became plain and basic. On the other hand printing it using Opera went slightly different. It rendered all background colours (header, th,..), but it didn't display the background images.We could have used @print to fix that, but in all cases essential elements of the page were fully visible and the background images will cost alot of ink to print.
