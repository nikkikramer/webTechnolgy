  • Usability

    • Usability is an important aspect to consider in any webpage, that's why we made sure that our one-page website also follows the usabilty guidelines. Our page Shows the company logo in a reasonable size and noticeable location. It has an about us section. For easier user experience we have made a navigation bar with 3 buttons. Accessibility highlights

As discussed between us, we find that the most relevant accessibility guidelines our page follows are:

  • 2.1 Keyboard accessible

    • Our page is fully accessible using the keyboard. So without having a working mouse you can access all elements using keyboard keys like the tab or arrows buttons. In addition, no functionality is lost since due to our keyboard accessibility, the form can be also be used with solely the keyboard to submit and reset products in the database. The main focus of our website is the products, so when our form would not be keyboard accessible, the website would not work properly as we envisioned.

  • 2.4 The page is navigable

    • Our website is provided by a fully working navigation bar. The user can use it to directly move to the most important sections or parts of our page regardless of what section you are in at that time. Therefore we think that it is one of guidelines our website adhires the most.

Guidelines Sub-guideline Evaluation
1.1 Text Alternatives 1.1.1 Non-text Content
  • Controls, Input:The website makes use of different type of user input's (text,number,url) where the names and id's of these user input tags describes the purpose of the user input which is asked. Every input field is also presented with a placeholder value which tells the user what kind of input is expected.
  • Time-Based Media: The webpage has a media file/video-only file present on the webpage. Nevertheless, this video is purely for visual design an does not have any meaning of the contents of the webpage. Therefore no text is accompanied with this video since it has not got any meaning.
  • Test: Guidline does not apply since, the webpage doesn't contain a test.
  • Sonsory: As stated in guidline 1.3.3 the webpage does not have any sensory characteristics, therefore the guidline does not apply.
  • CAPTCHA: Guidline does not apply since, the webpage doesn't contain a CAPTCHA.
  • Decoration, Formatting, Invisible: The webpage consists of a video in the 'about us' section which is pure decoration. The website fails this guidline since this video is not implemented in such a way that it can be ignored by assistive technology at the moment.
1.3 Adaptable 1.3.1 Info and Relationships The website does not use any auditory formatting therefore the information and relationships that are implied by auditory formatting are preserved. In addition the visual formatting also preserves the information and relationships, when one would remove the stylesheet from the web document the contents of the website does not change, thus it can be said that the information, structure and relationships conveyed through presentation are available in text.
1.3.2 Meaningful Sequence The website consists of a meaningful sequence. When one visites the webpage one would first come across the webpage title page, then one would come across the section describing what the webpage is about, following are the products available on the website. Thus it can be said that the content are presented in a menaningful order. In addition, if one would delete the CSS style sheet from the webpage the webpage would still display the page's content in the correct order.
1.3.3 Sensory Characteristics This website has no sensory characteristics, it does not describe any controls solely via references or auditory cues.
1.4 Distinguishable 1.4.1 Use of Color If one were to remove all the colors from the website the substantive content would not change, thus the information on the website is not conveyed through color differences. This ensures that users who cannot see color can still perceive the information on the website just as good as users who can see color.
1.4.2 Audio Control This doesn't apply to the website since there are no audio fragments available on the website.
2.1 Keyboard Accessible 2.1.1 Keyboard For this website, the entire content of the page is operable thorugh a keyboard or keyboard interface. Using keyboards keys like the tab, arrows, enter, etc a person with no vision who cannot use a mouse is still available to browse through the websites content and use the user input features to add certain products to the database.
2.1.2 No Keyboard Trap The form on the website can still be updated/reset using only the keyboard. Just like the links to other sections of the webpage and links to other websites which can be used solely with the keyboard. With the use of the keyboard a user is available to see and use the whole page as one would who has for example a mouse and keyboard available.
2.4 Navigable 2.4.1 Bypass Blocks The website has a navigation bar at the top of the document, which makes it easier for users to directly go the desired section. This purpose makes it easier to reach the main content of the website quickly and easily. Nevertheless, this navigation bar is only available when a user is at the top of the page. Therefore, the mechanism of the navigation bar is not always available to bypass blocks of content and therefore this guidline is failed.
2.4.2 Page Titled This website has headers for every section which describes the sections content. The title/name of the webshop can be found on the tab containing the page, which makes the title of the page easily availabble to the users for identifying the page.
2.4.3 Focus Order A user can find their way through the website in a sequential and meaningful order. The focus order follows a sequence that preserves the meaning and functionality of the page. The order in which the website is presented has meaning since first the user sees the title of the webpage, then the 'abous us' section which describes what kind of website the user is on. Following this the user is provided with products. This makes our website have a meaningful order. As described in point 2.4.2 the user can also navigate to every part of the website using the 'tab' key and use every function including forms and links using keyboard keys.
2.4.4 Link Purpose (In Context) Every link on this website describes where the user would be redirected to. The navigation bar in the top of the document describes which section there are directed to. The same goes for the links directed to the mobile providers, where the names describe where they are redirected to and hence their purpose.
